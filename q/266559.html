If "better" means better code in terms of readability and maintainability, you can use a regex like (...)(...)(.+) and to replace it with ($1) $2-$3.

This has the benefit that you can immediately tell that the pattern is going to take the first three characters as one group, the second three characters as the second group, and the rest as the third group. There's also less clutter in the "replace with" pattern which makes it pretty readable as well.

Finally, should you need to adjust the pattern, you can add or remove dots in any of the groups individually. Say that you discover that while the phone numbers are all US numbers but actually have a +1 country code prefix, and you need to extract that as part of the are code, you would only need to add two dots in the first group, like this:

(.....)(...)(.+)

The "replace with" pattern would not need changes.

That would be much less prone to mistakes than editing the numbers in the f"() in the question.

This is admittedly a contrived example. Any real phone number formatter would have to include many checks and additional criteria, using \d instead . and so on.
