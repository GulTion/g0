ＦＩ⟦θηθη⟧«

Loop over the width and height twice, alternating between them.

§; ↨υ¹

Output a space if the sum of the previous dimensions is odd but a semicolon if it is even.

⊞υι

Push the current dimension to the predefined empty list.

×;ι

Output that many semicolons.

↷

Pivot 90° so that the next iteration draws the next side of the rectangle.

Making the numbers of semicolons in the sides be even was harder than I expected - making them odd would have been 3 bytes shorter.

(Would have saved 4 bytes to input the width and height as a tuple instead of separate values.)
