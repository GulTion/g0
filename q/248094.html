Deobfuscated:

# Input arrives in $_, pre-trimmed to just digits

# Convert each digit into a pattern of bits corresponding
# to the segments that are turned on. The order of
# bits doesn't matter as long as they're
# consistent, so I've chosen an order that
# requires the fewest (octal) digits here.
# (this modifies $_ in-place)

tr/0-9/\167\3\136\37\53\75\175\7\177\77/;

# make a copy with the starting blank included;
# keep both in the predefined @_ array

@_ = ($_, "\0$_");

# (Now when we perform bitwise operations on these
# two values, it will apply the operation to each
# pair of consecutive bytes.)

# Do the following forwards and backwards to get
# both the "turned on" and "turned off" counts.

# (Note: loop iterator is $_, since we no longer need
# it to hold the input.)

for (0,1) {

    print unpack "%b*",  # count & print the bits
                 $_[$_] & ~ $_[!$_]
                        # !$_ changes 0 to 1 and 1 to 0
                        # (B & ~ A) tells you the bits that aren't
                        # set in A that are set in B

}

How?

Given the segment bitmask \$x\$ of the previous digit and the segment bitmask \$q\$ of the new digit, we compute \$k=x\operatorname{XOR}q\$ and use the helper function \$g\$ to update the off and on counters which are stored in the array \$o[\:]\$:

g = k =>       // k = result of the XOR between the 2 bitmasks
k &&           // stop when k = 0, which actually happens because
               // of arithmetic underflow because we use k / 2
               // rather than k >> 1 to save a byte
g(             // otherwise, do a recursive call:
  k / 2,       //   pass k / 2 for the next iteration
  o[q & 1] +=  //   select the counter according to the LSB of q
    k & 1,     //   increment it if the LSB of k is set
  q /= 2       //   divide q by 2
)              // end of recursive call
