How?

Similar to other regex based answers. Things that may or may not be Python specific: re.sub only finds non-overlapping matches. Therefore we use a look ahead assertion for the second copy of the repeat element so it remains available for further matches. *? is a non greedy version of *. Here it makes sure the match stretches only to the first reoccurrence of the repeat element.

k=Rv h'

ysk$k<>yS(hdS<*k)<>k

Attempt This Online!

Explanation

We are using a parser to do this ysk will repeatedly apply a parser until reaching a fixed point, so we just need to write a parser which does one step of the task.

As I point out in my comment it doesn't matter if you prioritize loops by their start or end location.

So we prioritize by the start. Parsers by default prioritize parsing more than less, so we create the parser k=Rv h' which acts as h' but with inverted priority.

To do this we start with a k, which will parse up to the start of the loop. Then the loop itself is yS$hdS<k, i.e. parse a character, parse some number of characters, parse the same character again returning only the last parse. Then we have k just to complete the rest of the string, the inverted priority doesn't matter here.

Reflection

There's definitely some room for improvement on hgl that we see in this answer.

  • This would be shorter with a regex if hgl flavored regexes could handle back-references. Implementing that is a big undertaking, but I always have to mention it.
  • Since I am defining k here, there's probably good cause to have Rv h' as a built-in, and might as well add Rv h_ too.
  • There could be a function to sandwich a parser between two copies of another parser. Basically 
    f x y = x <> y <> x
    And it should probably have preloaded variants for h' and h_.
