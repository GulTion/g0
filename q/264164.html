Cracks Sisyphus's, 1 byte

b'' and b'\x00'

I'm not sure if this is the intended solution but I think there's a strong argument for its validity. This solution abuses the fact that python lets us have null bytes inside of strings, but hmac (sometimes) ignores them. If you're hung up on the fact that one is the empty string then the crack could be b'a' and b'a\x00'. I would also point out that b'a', b'a\x00a', and b'aa' all produce different results, so obviously hmac can do something with null bytes, it just doesn't when they are at the end of a string.
